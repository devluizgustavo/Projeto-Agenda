Nesta aula iremos fazer os models ---> Pasta no qual irá ficar o banco de dados

Pra isso precisamos fazer a conexão com o banco.

1 - Uma boa prática de programação é NÃO deixar a senha do DB exposta...pra isso iremos instalar uma dependência chamada "dotenv"

Comando --> npm i dotenv

Dotenv --> Irá criar variáveis de ambientes aqui do nosso computador, e podemos remover essas variáveis para não ficar alocada ao projeto

Pra usar dotenv precisamos criar um arquivo chamado .env

Dentro deste arquivo só ira ter chave=valor... Não é javascript, é apenas para criar variáveis "ocultas"

Depois disto no arquivo do servidor vamos requerir o módulo, e colocar o seguinte comando para chamar a variável

Comando --> process.env.nome_variável

Após isso, vamos fazer o arquivo .gitignore --> Colocar o nome do arquivo .env dentro, para ele ser IGNORADO quando o projeto subir para o github
